01

Overview

Plain English

MergePDF is a browser-based utility. The core merge action happens locally in your browser, which means the app does not need to upload your PDF contents to complete the job.

This privacy policy explains what information is and is not handled when you use MergePDF, whether from a local file, a hosted static site, or a deployed copy using the same front-end code.

If you are using a customized or self-hosted version of this project, your deployment choices may change some details below, especially around hosting logs and third-party asset loading.

02

Files and document content

What matters most

Your PDF contents stay on your device during the merge process. The application combines them in browser memory and gives you a local download.

MergePDF does not require an account, does not ask you to create a profile, and does not intentionally transmit the contents of your selected PDF files to a server as part of the merge workflow.

Files you add are handled inside the browser session so they can be read, ordered, merged, and exported. Once you close the tab or clear the session, that in-memory working state is removed from the page.

If a PDF is corrupted or password-protected, the file may fail to load locally and the tool may show an error. That error handling still happens client-side.

03

Technical website data

Even though the merge itself is local, visiting a web page still creates normal web requests. Depending on how MergePDF is hosted, those requests can expose standard technical data such as:

  • Your IP address
  • Browser and device information sent in request headers
  • Requested page URLs and timestamps
  • Basic server or CDN logs needed to deliver the static files
Why this is different from file uploads

These ordinary page-load requests help deliver the website itself. They are separate from the merge workflow and do not mean your PDF contents were uploaded.

This build also adds a restrictive in-page Content Security Policy and serves the PDF processing library from a local file, which reduces third-party script exposure during normal use.

If you open a local offline copy instead of a hosted deployment, server-side request logging may not apply. The details depend on how the page is being served.

04

Cookies

The current MergePDF build does not use an account system, does not include advertising scripts, and does not set app-specific tracking cookies for the merge workflow itself.

That said, standard browser behavior, hosting layers, or third-party asset providers may still involve caching or other browser storage depending on your environment and browser settings.

If you self-host the font files as well, you can reduce the remaining third-party asset requests related to page delivery.

05

Third-party services

The current front-end uses a very small number of third-party services:

  • Google Fonts for typography. Loading those fonts may send technical request data such as IP address and user agent to Google.

Google receives the usual information involved in serving static web assets. It does not need the contents of your PDF files to deliver those fonts.

Local PDF processing bundle

The pdf-lib script is now shipped with this app instead of being loaded from a runtime CDN, which removes one external script dependency from the page.

You can review the PDF library used by this tool at pdf-lib.js.org.

06

Your choices

You have a few practical ways to tighten privacy even further when using or deploying MergePDF:

  • Use a local copy of the page instead of a public hosted deployment.
  • Self-host the font files so the page no longer depends on Google Fonts.
  • Use browser settings or extensions that limit external requests.
  • Clear your browser session after finishing sensitive document work.

If you are publishing your own version of this tool, you should also review your hosting provider's logging behavior and update this policy to match your exact deployment.

07

Policy changes

This policy may be updated if the front-end changes, if new third-party services are introduced, or if the hosting setup changes in a way that affects privacy expectations.

When that happens, the "Last updated" date at the top of this page should be revised so readers can see when the policy was last reviewed.

08

Contact

Questions about privacy, deployment, or data handling can be sent to privacy@mergepdf.com.

If you are reusing this project for your own deployment, replace that address with your real support or privacy contact before publishing.

What this means in practice

MergePDF is much more private than upload-based tools, but not magically request-free. The browser still has to fetch page assets unless you self-host everything.
